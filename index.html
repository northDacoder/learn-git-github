To get started using Git

Download git for OSX

Download git for Windows

Download git for Linux

Create a new repository:

Create a new directory, cd into the directory & run the command

$ mkdir project
$ cd project
$ git init

This will initialize Git version control in your current directory & allow you to start using the git commands

Clone an existing repository

Create a working copy of local repository

$ git clone /my/files/arent/organized/very/well

To create a local copy from a remote repository

$ git clone username@host:/server/projects/new-repo

Add & commit

To propose changes to be staged for your next commit

$ git add FILENAME
$ git add *

To actually commit these changes to the current HEAD

$ git commit -m "commit message"

This commits the file to the HEAD, but it is not in your remote repository yet

Now that your changes are in the HEAD of your local working copy, you can push those changes to your remote repository

$ git push origin master

You can also change master to whatever branch you want to push your changes to

$ git push origin feature-branch

If you have not cloned an existing repository from a remote server, then you will need to manually connect your local repo to the remote repo

First check to see if you already have a remote repo that you are connected to:

$ git remote -v

This will list all of the remote repositories you are able to push & pull code to

If you do not have a remote repo setup, you can manually add one by running this command:

https version - required password when pushing 

$ git remote add origin https://github.com/bob-dylan/project

ssh version - no password required when pushing

$ git remote add origin git@github.com:northDacoder/learn-git-github.git

To remove a remote repo

$ git remote remove origin https://github.com/bob-dylan/project
